# CitrusAgent 后端服务

广西橙子/柑橘产业智能助手后端，基于 **Python 3.11 + FastAPI + DeepSeek + Qdrant + MySQL** 构建，提供 RAG 本地知识库问答、意图路由、多场景对话服务。

## 核心功能

### 1. 登录认证

登录流程：**获取验证码 → 输入用户名+密码+验证码 → 验证通过返回 JWT Token → 后续请求携带 Token**

```text
前端请求 GET /auth/captcha
  → 返回 captcha_id + 验证码图片(Base64)
前端展示验证码图片，用户输入
前端请求 POST /auth/login
  → { username, password, captcha_id, captcha_code }
  → 验证码校验 → MySQL 用户验证 → 生成 JWT
  → 返回 { access_token, token_type, expires_in, user_info }
后续请求 Header: Authorization: Bearer <access_token>
```

**认证依赖注入**（在路由中使用）：

```python
from src.citrus_agent.api.deps import get_current_user, require_role

# 要求登录
@router.get("/protected")
async def protected(user: User = Depends(get_current_user)):
    ...

# 可选认证（有 token 返回用户，无 token 返回 None）
@router.get("/optional")
async def optional(user: User | None = Depends(get_current_user_optional)):
    ...

# 要求 admin 角色
@router.get("/admin-only", dependencies=[Depends(require_role("admin"))])
async def admin_route(user: User = Depends(get_current_user)):
    ...
```

**权限预留接口**：

| 方法 | 说明 | 当前实现 |
|------|------|----------|
| `check_permission(user, role)` | 角色层级权限检查 | admin(3) > user(2) > guest(1) |
| `check_resource_permission(user, resource, action)` | 资源级细粒度权限 | admin 全部权限，user 读写，guest 只读 |
| `require_role("admin")` | 路由级角色依赖工厂 | 用于 `dependencies=[Depends(...)]` |
| `create_user()` | 管理员创建用户 | 仅 admin 可调用 |

> 后续可扩展为 RBAC/ABAC 权限体系，当前验证码存储为内存字典，可替换为 Redis。

### 2. 三路对话与意图路由

对话模块提供三个独立的 SSE 流式入口，每个入口对应不同的 AI 助手身份和 Prompt 模板：

| 路由 | 入口 | AI 身份 | 默认 kb_id |
|------|------|---------|-----------|
| `general` | `/chat/general` | YAGO intellect 通用问答助手 | 1 |
| `agri` | `/chat/agri` | YAGO intellect 农业知识 AI 助手 | 2 |
| `diagnose` | `/chat/diagnose` | YAGO intellect 病虫害诊断助手 | 3 |

**对话流程**：

```text
用户提问
  → 是否指定 kb_id？
    → 是：直接检索指定知识库
    → 否：调用 IntentRouter.classify() 做意图判断
      → LLM 返回 { need_rag, kb_id, reasoning }
      → need_rag=true 且 kb_id 不为空 → 检索对应知识库
      → need_rag=false → 直接对话（不检索）
  → 有检索结果？
    → 是：使用 RAG Prompt（含知识片段）+ LLM 流式生成
    → 否：使用 Direct Prompt（无知识片段）+ LLM 流式生成
  → SSE 流式输出
    → data: {"delta": "文本片段"}
    → data: {"delta": "", "done": true, "from_knowledge_base": bool, "kb_id": int, "reasoning": "..."}
```

**Prompt 模板体系**：每个路由都有 Direct Prompt（不检索时）和 RAG Prompt（检索后）两套模板，均包含输出格式约束（HTML 标签输出）和防幻觉规则。

### 3. 文档入库流程

```text
后端接收上传文件
  → 校验文件类型（ALLOWED_EXTENSIONS）
  → 保存文件到磁盘（按年月分目录：uploads/YYYY-MM/uuid.ext）
  → 计算文件 SHA256
  → MySQL 创建 document 记录（status=processing）
  → 组装 DocumentEntity
  → 调用 RAGApiBridge.ingest_document()
    → 独立 RAG 模块可用时：调用 RAGApi.ingest_document()
      → DocumentParser 解析文档
      → KnowledgeIngestor 编排入库
        → 文本分块（固定长度 + 重叠）
        → EmbeddingProvider 生成向量
        → QdrantStore 写入向量（先删旧向量再写入新向量）
    → 独立 RAG 模块不可用时：回退到内置简化实现
  → 根据 result.success 更新 MySQL
    → 成功：status=ready，写入 chunk_count
    → 失败：status=failed，写入 error_message
```

**BGE-M3 Hybrid 入库**（可选）：

```text
RAGApi.ingest_document_with_bge_m3()
  → HybridKnowledgeIngestor 编排入库
    → 文本分块
    → BgeM3ApiEmbeddingProvider 生成 dense + sparse 向量
    → QdrantStore 写入 hybrid collection（dense + sparse 双向量）
```

### 4. RAG 模块架构

```
                    ┌─────────────────────┐
                    │    RAGApi (门面)      │
                    │  rag/rag_api.py      │
                    └──────┬──────────────┘
                           │
              ┌────────────┼────────────┐
              │            │            │
    ┌─────────▼──┐  ┌──────▼──────┐  ┌─▼──────────────┐
    │ Knowledge  │  │  Hybrid     │  │  CitrusRetriever│
    │ Ingestor   │  │  Knowledge  │  │  rag/retriever  │
    │ (dense)    │  │  Ingestor   │  │                 │
    └─────┬──────┘  │  (hybrid)   │  └────┬────────────┘
          │         └──────┬──────┘       │
          │                │              │
    ┌─────▼────────────────▼──────────────▼─────┐
    │         QdrantStore + EmbeddingProvider    │
    │         vectorstores/                      │
    └───────────────────────────────────────────┘
```

- **RAGApi**：后端调用 RAG 能力的统一门面，封装入库/删除/检索
- **KnowledgeIngestor**：dense-only 入库编排器，使用单向量
- **HybridKnowledgeIngestor**：BGE-M3 hybrid 入库编排器，使用 dense + sparse 双向量
- **CitrusRetriever**：知识库检索器，问题 → embedding → Qdrant search → 返回 top_k 个 chunk
- **RAGApiBridge**：桥接层，独立 RAG 模块可用时调用 RAGApi，不可用时回退到内置实现

### 5. RAG 检索增强生成（旧版服务）

`rag_service.py` 中的 `RAGService` 提供了另一套 RAG 流程（供 `chat_service.py` 使用）：

核心流程：**用户提问 → 查询向量化 → Qdrant 向量检索 → 构建 Prompt → LLM 生成回答**

- 严格基于知识库回答，知识库以外的内容拒绝回答
- 回答必须标注参考资料出处（文档标题 + 相似度分数）
- 相似度阈值可配置（默认 0.7），低于阈值不返回
- 检索时可传 `kb_id` 限定知识库范围，避免跨库串数据
- 支持流式和非流式两种生成方式

### 6. 多轮对话

- 通过 `conversation_id` 关联会话上下文
- 默认保留最近 10 轮对话历史
- 支持追问和引导式交互
- 会话存储在内存中（后续可替换为 Redis/DB）

### 7. 文档切片策略

当前使用固定长度 + 重叠的字符切片方式：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `CHUNK_SIZE` | 500 | 切片最大字符数 |
| `CHUNK_OVERLAP` | 80 | 相邻切片重叠字符数 |

切片流程：文本清洗 → 按固定长度切分 → 相邻切片保留重叠 → 生成 chunk_id（格式: `{kb_id}:{document_id}:{chunk_index}:{sha256_hash[:16]}`）

### 8. 工具调用

当前已注册工具：

| 工具名 | 说明 | 参数 |
|--------|------|------|
| `fertilizer_calculator` | 肥料用量计算 | `area`（面积/亩）、`crop_type`（作物类型）、`fertilizer_type`（肥料类型） |

肥料计算器支持的作物：柑橘、番茄、辣椒、白菜、黄瓜、水稻
肥料计算器支持的肥料类型：复合肥、尿素、钾肥、磷肥

新增工具只需继承 `BaseTool` 并调用 `ToolRegistry.register()`：

```python
from src.citrus_agent.tools.base import BaseTool, ToolRegistry

class MyTool(BaseTool):
    @property
    def name(self) -> str:
        return "my_tool"

    @property
    def description(self) -> str:
        return "工具描述"

    @property
    def parameters_schema(self) -> dict:
        return {"type": "object", "properties": {...}}

    async def execute(self, **kwargs) -> dict:
        return {"success": True, "result": ...}

ToolRegistry.register(MyTool())
```

### 10. 流式输出

对话接口统一使用 SSE 流式响应。响应格式：

**聊天路由（chat.py）**：

| 事件 | 格式 | 说明 |
|------|------|------|
| 文本片段 | `data: {"delta": "文本"}\n\n` | LLM 生成的增量文本 |
| 结束事件 | `data: {"delta": "", "done": true, "from_knowledge_base": bool, "kb_id": int, "reasoning": "..."}\n\n` | 生成完成，携带元信息 |

**RAG 服务（rag_service.py）**：

| type | 说明 |
|------|------|
| `source` | 参考资料来源 |
| `content` | 回答文本片段 |
| `tool` | 工具调用通知 |
| `done` | 生成完成 |

## 支持的文件类型

| 格式 | 扩展名 | 解析方式 |
|------|--------|----------|
| PDF | `.pdf` | pypdf（按页解析） |
| Word | `.docx` | python-docx（段落+表格） |

## Embedding 提供者

| 提供者 | 配置值 | 说明 | 向量维度 |
|--------|--------|------|----------|
| API (DashScope) | `api` / `openai` / `dashscope` | 阿里云 DashScope OpenAI 兼容模式，text-embedding-v4 | 1024 |
| Local BGE | `local_bge` / `bge` / `local` | 本地 sentence-transformers 模型 | 由模型决定 |
| BGE-M3 Server | - | 独立部署的 BGE-M3 服务，返回 dense + sparse | 1024 |
| Fixed Debug | `fixed` / `debug` | 测试用固定维度伪向量 | 可配置 |

## 应用生命周期

启动时自动执行：

1. **MySQL 数据库初始化**：自动建表（`init_db()`）
2. **默认管理员创建**：检测 admin 用户不存在时自动创建
3. **Qdrant 连接初始化**：确保 Qdrant 可连接
4. **工具注册**：导入 `fertilizer_calculator` 模块触发注册
5. **RAG 桥接初始化**：尝试连接独立 RAG 模块，失败则回退内置实现

关闭时自动执行：

1. **关闭 MySQL 连接池**：释放所有数据库连接
